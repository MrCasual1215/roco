# 任务策略记录

## 1. Sort Cubes / `sort`

### 目标

将方块放到对应目标区域或面板上。

### 推荐策略

1. 识别每个 cube 对应的目标；
2. 分配给最近或可达机器人；
3. 机器人先 `PICK`，下一步 `PLACE`；
4. 其他机器人 `WAIT`，减少碰撞。

### 常见失败

- 物体名写错；
- 同一物体被多个机器人同时抓；
- 放置目标错误；
- 机器人不可达。

## 2. Sweep Floor / `sweep`

### 目标

机器人用扫帚和簸箕将垃圾扫入簸箕，再倒入垃圾桶。

### 推荐策略

1. 一个机器人负责簸箕；
2. 一个机器人负责扫帚；
3. 先移动簸箕到垃圾附近；
4. 再执行 `SWEEP`；
5. 最后执行 `DUMP`。

### 常见失败

- 簸箕没到位就扫；
- 扫帚路径碰撞；
- DUMP 位置不对；
- 两机器人同时移动导致碰撞。

## 3. Arrange Cabinet / `cabinet`

### 目标

多个机器人打开柜门，从柜子中取杯子并放到正确杯垫。

### 推荐策略

1. 先 `OPEN` 左右柜门；
2. 保持门打开；
3. 第三个机器人取杯子；
4. 逐个杯子放置到对应 coaster。

### 常见失败

- 没开门就取杯子；
- 门关闭或挡住取物路径；
- 机器人在柜门附近碰撞；
- 杯子与杯垫匹配错误。

## 4. Make Sandwich / `sandwich`

### 目标

两个机器人按正确顺序堆叠三明治食材。

### 推荐策略

1. 明确食材堆叠顺序；
2. 一次只让一个机器人放置食材；
3. 另一个机器人等待或准备下一个食材；
4. 避免同时往中心区域放东西。

### 常见失败

- 食材顺序错误；
- 放置位置偏移；
- 两机器人同时进入堆叠区域；
- 抓取后未正确释放。

## 5. Pack Grocery / `pack`

### 目标

两个机器人把桌面杂货放进箱子。

### 推荐策略

1. 一次打包一个物品；
2. 优先选择容易抓取、路径短的物品；
3. 使用高一点的路径点绕开箱子边缘；
4. milk、cereal 等较高物体需要注意放置高度；
5. 其他机器人 `WAIT`，减少碰撞。

### 注意

代码中 `pack` 会强制 `action_and_path`，模型需要输出 `PATH`。

### 常见失败

- PATH 格式不合法；
- 物体放置高度太低导致碰撞；
- 物体与箱子边缘碰撞；
- 两机器人路径交叉。

## 6. Move Rope / `rope`

### 目标

两个机器人抓住绳子两端，协同将绳子越过墙并放入槽中。

### 推荐策略

1. 两机器人分别抓绳子两端；
2. 同步抬高；
3. 同步越过墙；
4. 同步下降到槽中；
5. 保持两端路径平滑且相对距离稳定。

### 注意

代码中 `rope` 会强制 `action_and_path`，模型需要输出 `PATH`。

### 常见失败

- 两端动作不同步；
- PATH 不平滑；
- 路径点太低撞墙；
- IK 失败；
- 两机器人互相碰撞。
